* Presentation simple du Green Code : 
   
Le Green Code (ou Green Coding) désigne une façon de concevoir et écrire du code informatique plus respectueux de l’environnement.
L’idée principale est simple : un logiciel mieux conçu consomme moins de ressources, donc moins d’énergie, ce qui réduit son impact écologique.

- Coder de manière responsable

- Consommer moins d’énergie

- Ecrire du code plus performant et plus propre

   Les principes fondamentaux du Green Code

1 - Utiliser moins de ressources :

- Éviter les calculs inutiles.
- Ne charger que les données nécessaires.
- Libérer correctement la mémoire.

2 - Optimiser les algorithmes :

- Choisir un algorithme plus rapide.
- Réduire les boucles imbriquées.
- Éviter les traitements répétitifs.

3 - Réduire les appels réseau et base de données :

- Regrouper les requêtes.
- Mettre en cache quand c’est possible.
- Éviter les appels inutiles.

4 - Écrire du code simple et lisible (KISS) :

- Un code clair est plus facile à optimiser.
- Moins de bugs = moins de correctifs = moins de ressources.
